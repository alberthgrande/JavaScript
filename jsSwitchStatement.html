JavaScript Switch Statement

In programming, there are times that we need multiple if statements.
In this example, the progam prints the color of the given fruit by checking what fuit is specified.

var fruit = "mango";
if(fruit == "apple") {
document.getElementById("demo").innerHTML = "The color is red.";
}
if(fruit == "strawberry") {
document.getElementById("demo").innerHTML = "The color is pink.";
}
if(fruit == "mango") {
document.getElementById("demo").innerHTML = "The color is yellow.";
}

Sample output:

As you may notice, the code above looks ineffecient.
Instead of writing multiple if statements, we can shorter the code above woth switch statement.

var fruit = "mango";
switch(fruit) {
case "apple":
document.getElementById("demo1").innerHTML = "The color is red.";
break;
case "strawberry":
document.getElementById("demo1").innerHTML = "The color is pink.";
break;
case "mango":
document.getElementById("demo1").innerHTML = "The color is yellow.";
break;
}

Sample output:

Note! The first and second example work the same way, however, the second example is cosidered good practice and more effecient.

The switch Statement

The switch statement matches an expression agains the specified values in the case clauses.

Syntax:

switch(expression) {
case value1:
// code blocks to be executed
break;
case value2:
// code blocks to be executed
break;
case value3:
// code blocks to be executed
break;
}

The switch statement evaluates the expression to value1 first, then to value2 and so on.

Here is a basic example:

var city = "Mumbai";
switch(city) {
case "Washington, D.C.":
document.write("It is the federal capital city of the United States.");
break;
case "Mumbai":
document.write("It is capital city of India.");
break;
case "Abuja":
document.write("It is the capital city of Nigeria.");
break;
}

Note! The evaluations are sensitively cased.

The break Keyword

The break keyword comes after the code blocks of each case calause.
Although it is optional, it is highly recommended.
The role of the break keyword is to stop the execution of the the switch statement when an evaluation already match a case.

var fruit = "apple";
switch(fruit) {
case "apple": // fruit is apple so the code below will now run
document.write("The color is red.");
// no break is given here
case "strawberry": // this case and the following one will run
document.write("The color is pink.");
case "mango":
document.write("The color is yellow.");
}
// following statements will be delayed a bit

Sample output:

In the example above, although the expression already matched the first value.
The second and third case still runned, even though there is no longer need to.
Instead of being executed right after the first
case, the following statements after the
switch will have to wait for the second and third
cases to finish running.

The default Clause

The default clause is executed if the expression does not match any case.

Here is simple example:

var fruit = "lettuce";
switch(fruit) {
case "apple":
document.write("The color is red.");
break;
case "strawberry":
document.write("The color is pink.");
break;
case "mango":
document.write("The color is yellow.");
break;
default:
document.write("That is not a fruit.");
}
